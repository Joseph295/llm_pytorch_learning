## 8.1 来龙去脉：注意力之外，还缺四块拼图

第 7 章的多头注意力单独拿出来是训不出语言模型的。缺的四块，每块都有明确的"不装会死"的理由：

**拼图一：位置信息**。注意力是**置换等变**的——把输入 token 打乱，输出以同样的方式打乱（QKV 全是逐位置线性变换 + 无序的加权和，没有任何部件"知道"顺序）。"猫追狗"和"狗追猫"在它眼里是同一个集合。必须显式注入位置（8.2-①）。

**拼图二：FFN**。注意力负责"搬运信息"（从别的位置取东西），但对取来的信息做**非线性加工**的能力很弱（softmax 加权和本质上还是插值）。FFN——逐位置的两层 MLP——补上加工能力。可解释性研究给了它更响亮的名字：**知识存储**（key-value memory：第一层匹配模式，第二层写出关联内容）。它占 Transformer 约 2/3 的参数不是浪费，是主力仓库。

**拼图三：残差连接**。没有它，堆 12 层就训不动（第 3 章你画过残差的梯度双路图）。现代视角把它升格为**残差流（residual stream）**：一条贯穿全网的信息主干道，每层只是从主干道"读取 → 加工 → 写回增量"。这个视角能解释很多设计（为什么删掉中间某层模型只是变笨不是崩溃——主干道还在）。

**拼图四：归一化**。深网络的激活方差会逐层漂移（放大→溢出，缩小→信号消失）。归一化在每层入口把分布拉回标准位。**放哪里**曾是大问题：原始 Transformer 的 Post-Norm（残差相加之后归一化）深了很难训，要靠精细 warmup 伺候；GPT-2 起改用 **Pre-Norm**（进入子层之前归一化，残差主干道上没有任何东西挡路）——梯度可以沿主干道无阻尼直达任何一层，深模型的训练稳定性大幅改善，成为 LLM 的绝对主流。

## 8.2 核心原理

### ① 位置编码的演进：为什么 RoPE 赢了

**学习式绝对位置（GPT-2）**：`pos_embed = nn.Embedding(max_len, d)`，加在 token embedding 上。简单有效，两个硬伤：训练长度之外没有定义（T=1024 训练，第 1025 个位置的 embedding 从没学过——直接越界报错或胡输出，易错点⑤）；且"绝对位置"本身不是好的归纳偏置——语言关心的多是**相对**关系（"形容词修饰下一个名词"不关心它们在第几位）。

**正弦位置编码（原始 Transformer）**：固定公式，理论上无限长。但相对性只是隐式的，实践并不比学习式强多少。

**RoPE（旋转位置编码，2021）**：现代 LLM 的统一答案（LLaMA/Qwen/DeepSeek 全用）。思想的精髓：**不往 embedding 里加位置，而是把位置变成 Q、K 向量的旋转角**。把 d 维向量看成 d/2 个二维平面，位置 m 的向量在第 i 个平面上旋转角度 `m·θᵢ`（θᵢ 按维度递减，像一组不同快慢的时钟指针）。魔法在点积上：

```
旋转后的 q_m · 旋转后的 k_n = 只依赖 (m−n) 的函数
```

（二维旋转矩阵的性质：`R(a)ᵀR(b) = R(b−a)`。）**注意力分数天然只看相对距离**——这正是语言要的归纳偏置。额外收益：外推性好（配合插值技巧可把 4K 训练的模型拉到 32K+ 上下文，第 10 章）、不占参数、作用在每层（而不是只在入口加一次）。实现要点：施加在**拆头之后的 Q、K** 上（不动 V——V 是"内容"，位置只该影响"检索匹配"），预计算 cos/sin 表存 buffer（第 4 章的知识落位）。

### ② Pre-Norm 的梯度论证 + RMSNorm 的减法

Pre-Norm 块的形状：`x = x + Attn(Norm(x)); x = x + FFN(Norm(x))`。对它求导：`∂x_out/∂x_in = I + ∂子层/∂x_in`——**恒等项 I 无条件存在**，梯度沿主干道无衰减直达（Post-Norm 的归一化挡在主干道上，恒等项被 Norm 的雅可比调制，深了就衰减）。代价：Pre-Norm 的残差流范数随深度缓慢增长（每层无脑相加），所以出口要一个 **final norm** 收尾（易错点④）。

**RMSNorm**：LayerNorm 做两件事——减均值、除标准差（再仿射）。RMSNorm 发现第一件事可以省：只除以均方根 `x / RMS(x) · γ`（连 bias 也不要了）。效果不掉、算力和参数都省——LLM 时代"能删就删"哲学的代表作。本章挑战题手写并实测。

### ③ FFN：4 倍宽的两层仓库

```python
h = W2( GELU( W1(x) ) )     # W1: d → 4d,  W2: 4d → d
```

为什么先放大 4 倍再收回：中间的高维空间是"模式检测器"的容量——W1 的每一行是一个待匹配的模式（key），GELU 做半开关，W2 的每一列是匹配后写回残差流的内容（value）。4 这个倍数是原始论文的经验值，被沿用成了行业默认（SwiGLU 变体用 8/3 补偿三矩阵，第 10 章）。**参数账**：FFN 每层 8d²，注意力每层 4d²——FFN 是参数大头，这决定了它也是算力大头（MoE 正是冲着它做稀疏化，第 10 章）。

### ④ 初始化：深度网络的方差纪律

残差流每层都在"加东西"：L 层之后输出方差 ≈ 各层写入方差之和。不管的话 48 层的 GPT-2 输出方差膨胀 ~L 倍。GPT-2 的对策（沿用至今）：**凡是"写回残差流"的投影（注意力的 W_o、FFN 的 W2），初始化标准差除以 √(2L)**（2L = 每层两次写入 × L 层）——各层写入的总和方差恰好归一。其余矩阵用标准的 0.02 或 1/√d。本章实验用第 4 章的激活统计器验证：不加缩放，深模型逐层方差爬升；加了，全程平稳。**这半行代码是"深模型训得动/训不动"的分界线之一。**

### ⑤ 组装 GPT 与参数量心算公式

```
tokens → Embedding(V, d) ─┐
                          ├→ [Block × L] → FinalNorm → lm_head(d, V)（与 embedding tie）
Block = x + Attn(Norm(x)) ┘
        x + FFN(Norm(x))
```

参数量公式（tie weights 后）：

```
每层: 注意力 4d² + FFN 8d² ≈ 12d²
总计: 12·L·d² + V·d (embedding) + 少量 norm 参数
```

心算演练——GPT-2 124M：L=12, d=768, V=50257 → 12×12×768² ≈ 85M，embedding 50257×768 ≈ 39M，合计 ≈ 124M ✓。**这个公式请练到 30 秒内完成**——面试现场估算、读论文时核对配置、规划显存（第 2 章的账）都靠它。

## 8.4 易错点清单

**① RoPE 施加的位置不对**
→ **正确**：拆头后的 q、k 上逐头施加；**不施加于 v**。施加在拆头前的整条 d_model 向量上（维度配对错乱）、或顺手也旋转了 v（位置污染内容），都是"能训但差一截"的静默错误。与官方实现对拍（实验里有）是唯一保险。

**② 学习式位置编码的长度越界**
→ **现象**：`IndexError: index out of range`（推理长度 > 训练 max_len）。
→ RoPE 无此问题（角度公式对任意位置有定义，虽然远超训练长度时质量下降——外推与插值话题见第 10 章）。

**③ 残差投影初始化忘缩放**
→ **现象**：层数 ≤ 6 时看不出来；12 层以上训练初期 loss 波动大、深层激活统计逐层放大（用激活统计器一看便知）。修正即 8.2-④ 的 /√(2L)。

**④ 忘了 final norm**
→ **现象**：Pre-Norm 模型能训，但 lm_head 吃到的残差流范数随深度膨胀，logits 偏大，学习率难调。一行 `self.norm_f = RMSNorm(d)` 的事。

**⑤ dropout 位置放错**
→ 惯例位置：注意力权重上（softmax 后）、每个子层写回残差流之前、embedding 之后。放在残差主干道上（`Dropout(x + sublayer)`）会随机切断主干道——伤害远大于正则收益。（预训练数据充足时现代 LLM 常干脆 dropout=0，微调小数据才开。）

**⑥ Norm 的 eps 拍脑袋**
→ RMSNorm/LayerNorm 的 `eps`（1e-5/1e-6）在 bf16 训练里不是装饰——太小会在低精度下失去保护作用（方差极小时除法爆大数）。跟随成熟配置（LLaMA 用 1e-5/1e-6），不要自己发明。

## 8.6 典型面试题

**Q1：Pre-Norm 和 Post-Norm 的区别？为什么现代 LLM 都用 Pre-Norm？**

> **参考答案**：Post-Norm：`Norm(x + Sublayer(x))`，归一化在残差相加后（挡在主干道上）；Pre-Norm：`x + Sublayer(Norm(x))`，主干道纯净。求导看差别：Pre-Norm 的层间雅可比含无条件的恒等项，梯度直达浅层，深模型免 warmup 技巧也稳；Post-Norm 深了梯度衰减，训练脆弱。代价：Pre-Norm 残差范数随层数增长（需 final norm），且有研究认为同深度下 Post-Norm 的表达效率略高——但训练稳定性的价值碾压。**加分点**：提到折中方案（Sandwich Norm、DeepNorm）和 final norm 的必要性。

**Q2：RoPE 的原理？为什么它对长上下文外推友好？**

> **参考答案**：将 Q/K 的每对相邻维度视为二维平面，按位置 m 旋转 m·θᵢ（θᵢ = 10000^(−2i/d)，多尺度频率）。旋转矩阵性质使 q_m·k_n 只依赖相对位移 m−n——注意力天然相对化。外推：绝对可学位置在超出训练长度处未定义；RoPE 角度处处有定义，配合位置插值（PI）/NTK-aware 缩放/YaRN 等技巧，可以低成本扩展上下文（把超界位置"压缩"回训练过的角度范围）。**加分点**：低频维度（转得慢的指针）承载长程信息，插值主要发生在这些维度；RoPE 只旋转 QK 不动 V 的原因（位置只应影响匹配，不应污染内容）。

**Q3：现场估算：LLaMA-7B（L=32, d=4096, V=32000）的参数量，并说明哪部分最大。**

> **参考答案**：主干 12Ld²（SwiGLU 略高，粗算够用）= 12×32×4096² ≈ 6.4B；embedding V·d = 32000×4096 ≈ 0.13B；合计 ≈ 6.6B ≈ 7B ✓。最大头是 FFN（每层 8d²+ 占主干 2/3）。**加分点**：由参数量立即推显存（×2 = 13GB 推理 fp16，×16 = 104GiB 训练状态，第 2 章的账）；vocab 增大（多语言 150K+）时 embedding 占比显著上升，tie weights 的价值随之增大。

**Q4：FFN 在 Transformer 里的作用是什么？如果把它删掉会怎样？**

> **参考答案**：逐位置的非线性加工与知识存储（key-value memory 视角：W1 行=模式检测，W2 列=关联内容写回）。删掉后模型退化为"注意力反复加权平均"——表达能力大幅受限（多层注意力的复合仍近似低秩变换），事实性知识没地方存。可解释性研究（知识编辑如 ROME）直接在 FFN 权重里定位并改写事实关联，是"FFN=知识库"的实证。**加分点**：注意力和 FFN 的分工——搬运 vs 加工；MoE 对 FFN 做稀疏化（它是参数与算力大头）而保留注意力稠密。

## 8.7 疑难杂症排查

**案例 1：12 层模型 loss 正常，24 层同配置发散**

深度敏感的病，三个检查位：① 残差投影初始化缩放（8.2-④）——用激活统计器沿层打印输出 std，逐层爬升即确诊；② Norm 遗漏（某个子层前的 Norm 被删/写错位置）；③ lr 没有随深度调低（深模型对 lr 更敏感，配合 warmup 拉长）。**方法论**：深度相关的问题，先看"逐层统计剖面"而不是盯 loss。

**案例 2：训练好好的，推理时输入稍长就崩/变差**

崩（报错）→ learned pos embedding 越界（易错点②）；不崩但质量骤降 → RoPE 超出训练长度的外推衰减（属预期行为，解法是第 10 章的插值方案），或 attention sink 类分布问题。区分手段：在训练长度内截断重试，恢复正常即长度问题实锤。

**案例 3：loss 降到某个值就纹丝不动，比预期高一截**

排查序：① 泄漏测试反着做——确认没有**过度屏蔽**（mask 写反：`masked_fill(mask)` 而不是 `masked_fill(~mask)`，模型只能看未来看不到过去，loss 会卡在接近均匀分布的水平）；② logits 尺度（忘 final norm / 忘 √d 缩放的注意力饱和）；③ 数据本身的熵下界（第 9 章讲：loss 的理论下界是数据的条件熵，不可能无限降）。**心算参考**：完全随机猜 V=50257 的词表，loss = ln(50257) ≈ 10.8；训练开局在 10~11 是正常起点。
