## 10.1 来龙去脉：从 GPT-2 到现代 LLM，改了什么、为什么

你的 miniGPT（第 8 章）已经比 GPT-2 现代（RoPE + RMSNorm）。但和 2024 年的 LLaMA-3/Qwen2.5/DeepSeek-V3 相比还差几步。这些改动**没有一个是为了炫技**，每个都对着一个具体的规模化痛点：

| 改动 | 解决的痛点 | 代价 |
|---|---|---|
| GQA/MQA | 推理时 KV cache 显存爆炸（第 7 章的账） | 轻微质量损失 |
| SwiGLU | FFN 表达力不足 | FFN 多一个矩阵 |
| MoE | 想要大参数量又不想要大算力 | 显存、通信、训练复杂度 |
| 长上下文（RoPE 插值） | 训练长度限死了可用上下文 | 长程精度略降 |
| QK-Norm 等稳定性 trick | 超大规模训练发散 | 微小算力 |

主线逻辑：**预训练时代竞争的是"每 FLOP 的智能"和"每 GB 显存的服务能力"**。这两条主线解释了几乎所有架构改动——省算力（MoE）、省显存（GQA）、省得起就换质量（SwiGLU）。

## 10.2 核心原理

### ① GQA/MQA：KV cache 的减负术

第 7 章挑战题你已经实现过 GQA，这里补齐"为什么"的全景。回顾推理的显存瓶颈：自回归生成要缓存每一步的 K、V（避免重算，第 16 章详解），KV cache 大小 = `2 × L × H × T × d_head × 并发数 × 字节`——**长上下文 + 高并发时它超过模型权重本身**。

- **MHA**（H 个 KV 头）：cache 最大，质量最好。
- **MQA**（1 个 KV 头，所有 query 头共享）：cache 缩小 H 倍，质量掉一点。
- **GQA**（G 组 KV 头，1<G<H）：折中，LLaMA-2 70B 起的标配（G=8）。

关键洞察：**query 头可以很多（表达多视角），但它们查询的 KV 可以共享**——检索的"视角"数和"被检索内容"的份数解耦。GQA 省的是显存与访存带宽（KV 读取是解码的带宽瓶颈），注意力的计算 FLOPs 基本不变（第 7 章挑战题的结论）。本章实验对比三者的 cache 大小与生成质量。

### ② SwiGLU：门控让 FFN 更聪明

miniGPT 的 FFN 是 `W2(GELU(W1 x))`。SwiGLU 换成**门控**结构：

```
FFN_SwiGLU(x) = W2( SiLU(W_gate x) ⊙ (W_up x) )     # ⊙ 逐元素乘
```

多了一个 `W_gate` 矩阵：`W_up x` 是"候选内容"，`SiLU(W_gate x)` 是"每个通道开多大"的**数据依赖门控**（不是固定的激活函数曲线，而是让输入自己决定放行多少）。这种乘法交互给了 FFN 更强的表达力（类似 LSTM 门控的精神）。

**为什么宽度变 8/3 d**：SwiGLU 有三个矩阵（gate/up/down）而普通 FFN 两个。为了保持总参数量与 4d 的普通 FFN 相当，中间维度设为 `(4d)×(2/3) = 8/3 d`——LLaMA 里你看到的 `11008 ≈ 8/3 × 4096` 就是这么来的（再向上取整到硬件友好的倍数）。本章实验手写 SwiGLU 替换 miniGPT 的 FFN。

### ③ MoE：稀疏激活，参数与算力脱钩

痛点：想让模型"知道更多"（更多参数）但不想每个 token 都付全部算力。MoE（专家混合）的答案——把 FFN 替换成 N 个并行的"专家 FFN"，每个 token 只**路由**到 top-k 个（通常 k=2）：

```
gate = softmax(W_router x)          # 每个 token 算出对 N 个专家的偏好
选 top-2 专家，加权合并它们的输出   # 只激活 2/N 的参数
```

DeepSeek-V3：671B 总参数，但每 token 只激活 37B——**推理算力相当于 37B 稠密模型，知识容量相当于 671B**。代价三重：① 显存要放下全部专家（第 2 章的账，671B 全参数状态得存）；② 专家分布在多卡时 token 路由产生 all-to-all 通信（第 13 章）；③ **负载均衡**——不加约束的话 router 会偏爱少数专家（"赢者通吃"），要加辅助损失逼平均。本章实验实现一个玩具 MoE，亲眼看负载失衡和均衡损失的效果。

### ④ 长上下文：RoPE 插值的魔法

miniGPT 的 block_size=256，超过就质量崩（第 8 章 RoPE 外推衰减）。真实模型要 128K+ 上下文，怎么做？核心技巧是 **RoPE 位置插值**：训练时模型见过位置 0~4095 的旋转角，想支持 32K 就把新位置"压缩"回训练过的角度范围——

- **PI（位置插值）**：位置 m 直接乘 4096/32768，线性压回 → 简单但高频信息受损；
- **NTK-aware**：不同频率维度不同缩放（高频少压、低频多压），保住近距离分辨率；
- **YaRN**：NTK 的精修版，配合少量长文本微调，成为事实标准。

思想都是"复用已学到的角度几何，把长序列的位置映射进去"。本章实验演示 PI 如何让 256-上下文的 miniGPT 处理 512 长度（虽然质量下降，但不再崩）。

### ⑤ 训练稳定性 trick 拾遗

超大规模（百 B 参数、万卡）训练容易发散，几个高频补丁：**QK-Norm**（对 Q、K 做 RMSNorm 再算注意力，压住 logits 爆炸，Qwen2/Gemma2 用）、**Z-loss**（惩罚 logits 的 log-sum-exp，防数值漂移）、**embedding/logit 缩放**、**更保守的 β₂ 和更长 warmup**（第 6 章）。这些在小模型上看不出必要性，规模上去才显形——知道它们存在，排查大规模训练发散时才有方向（第 15 章）。

## 10.4 易错点清单

**① GQA 的 KV 头复制用错算子**（第 7 章挑战题回顾）
→ `repeat` 真复制吃显存，应该用 `expand`（零拷贝）；且复制要在 RoPE **之后**（K 先旋转再铺开，否则各 query 头看到的位置几何不一致）。

**② SwiGLU 宽度没对齐，参数量暴涨**
→ 直接把三矩阵都设成 4d，参数量比 GELU 版多 50%。要用 8/3 d 保持相当（10.2-②）。对拍参数量（实验里有）。

**③ MoE 忘了负载均衡损失**
→ **现象**：训练 loss 正常，但少数专家承载了绝大多数 token（其余专家"饿死"，等效参数量骤减）。
→ **修正**：加辅助均衡损失（鼓励 router 分布均匀）；监控每个专家的 token 占比（实验里可视化）。

**④ MoE 推理显存按"激活参数"估算**
→ **现象**：以为 DeepSeek-V3 只用 37B 就只需 37B 显存 → OOM。
→ **原因**：激活的是算力，**权重要全部驻留**（哪个 token 路由到哪个专家事先不知道）。显存按 671B 算。

**⑤ 长上下文只改推理不改位置编码**
→ 把 block_size 从 256 改成 2048 直接推理，RoPE 的位置远超训练范围 → 输出崩坏。必须用插值（10.2-④）或长文本微调。

## 10.5 开源项目的最佳实践

**① LLaMA-3 / Qwen2.5 modeling 源码通读**
[modeling_llama.py](https://github.com/huggingface/transformers/blob/main/src/transformers/models/llama/modeling_llama.py) 现在你能逐行读懂：`LlamaAttention`（GQA + RoPE）、`LlamaMLP`（SwiGLU）、`LlamaRMSNorm`、`LlamaRotaryEmbedding`（含 scaling 配置）。对照你的 miniGPT，差异只在这几个零件的具体实现。**练习**：拿 miniGPT 的组件逐个替换成 LLaMA 版，你会得到一个能加载真实 LLaMA 权重的模型（本章挑战题）。

**② DeepSeek-V3 的 MoE 与 MLA**
[DeepSeek-V3 技术报告](https://github.com/deepseek-ai/DeepSeek-V3) 的两个创新：**MLA（多头潜在注意力）**——把 KV 压缩到低秩潜空间再缓存，KV cache 比 GQA 更省（GQA 的下一代）；**细粒度 MoE + 共享专家 + 无辅助损失的负载均衡**（用可学习 bias 替代辅助损失）。读它 = 看到 2025 年架构前沿。

**③ 配置驱动的架构选择**
transformers 的 config 里 `num_key_value_heads`（=num_heads 就是 MHA，=1 就是 MQA，中间是 GQA）、`intermediate_size`（FFN 宽度）、`rope_scaling`（长上下文策略）——一个架构族用配置参数覆盖多种变体。这是第 4/8 章"配置与结构分离"的极致：**同一份 modeling 代码，配置一变就是不同架构**。

## 10.6 典型面试题

**Q1：MHA、MQA、GQA 的区别？各自的 KV cache 大小和适用场景？**

> **参考答案**：区别在 KV 头数——MHA 每个 query 头独立 KV（H 个），MQA 所有 query 共享 1 个 KV，GQA 分 G 组（1<G<H）。KV cache ∝ KV 头数，MQA 最省（H 倍）质量略降，MHA 最好最费，GQA 折中（G=8 常见，省 H/G 倍）。场景：训练/质量优先用 MHA/GQA，推理服务/长上下文/高并发用 GQA/MQA。**加分点**：GQA 省的是访存带宽（解码是 memory-bound，第 16 章）和显存，FLOPs 基本不变；下一代 MLA（DeepSeek）用低秩压缩进一步减 cache。

**Q2：SwiGLU 为什么比 ReLU/GELU 好？为什么中间维度是 8/3 而不是 4？**

> **参考答案**：SwiGLU 引入数据依赖的门控（SiLU(W_gate·x) 逐通道调制 W_up·x），乘法交互增强表达力，实证上同参数量下困惑度更低。8/3：SwiGLU 有三个权重矩阵（gate/up/down）而普通 FFN 两个，为保持参数量与 4d 的两矩阵 FFN 相当，中间维取 4d×2/3=8/3 d。**加分点**：门控思想源自 GLU 家族（Shazeer 2020），本质是让网络自己学"开关"而非固定激活曲线；实际实现中间维还会向上取整到硬件友好的倍数（如 128 的倍数）。

**Q3：MoE 如何做到参数量和推理算力解耦？它的主要工程挑战是什么？**

> **参考答案**：FFN 换成 N 个专家，router 给每 token 选 top-k（如 2）个激活——算力只花在 k 个专家上（∝ 激活参数），但模型容量 ∝ 全部专家参数。DeepSeek-V3 671B 总参 / 37B 激活即此。挑战：① 显存要放下全部专家（激活的是算力不是显存）；② 专家并行下的 all-to-all 通信（token 要发到专家所在的卡，第 13 章）；③ 负载均衡（router 易偏科，需辅助损失或 bias 校正）；④ 训练不稳定（路由的离散选择）。**加分点**：细粒度专家 + 共享专家的现代设计；推理时的专家并行部署与 batching 策略。

**Q4：一个 4K 上下文训练的模型，如何扩展到 32K？有哪些方法，各自代价？**

> **参考答案**：核心是 RoPE 位置插值——把超训练范围的位置映射回已学的角度范围。方法：PI（线性压缩所有位置，简单但损高频分辨率）、NTK-aware（按频率差异化缩放，保近距离）、YaRN（NTK+温度调整+少量长文本微调，事实标准）。代价：纯插值不微调质量下降，长文本微调需要长序列训练数据和更多显存（O(T²)）。**加分点**：也有免训练的 StreamingLLM（attention sink + 滑动窗口）、以及架构层面的原生长上下文设计；评估长上下文要用 needle-in-haystack 等专门基准，困惑度不敏感。

## 10.7 疑难杂症排查

**案例 1：加载官方 LLaMA 权重到自实现模型，shape 全对但输出是乱码**

高发病因（都是"数值对但语义错位"）：① QKV 的拆头/合并顺序与官方不一致（官方可能是 `[head0_all_dims, head1_all_dims...]` 而你按 `[dim0_all_heads...]` 拆——权重排布约定不同）；② RoPE 的频率计算或施加方式有微妙差异（half-rotation 的维度配对方式：交错 vs 前后半分，LLaMA 和某些实现不同）；③ RMSNorm 的 eps 或是否上转 fp32 不一致。排查：逐层对拍中间激活（喂相同输入，比第一个 Norm 后、第一个注意力后…找第一个 diverge 的层）。**方法论**：加载外部权重出错，二分定位第一个数值发散的层。

**案例 2：MoE 训练 loss 正常但效果差**

查专家利用率：统计每个专家收到的 token 数，若高度倾斜（少数专家占 90%+），是负载失衡（易错点③）——加/调均衡损失。也可能是 top-k 太小（k=1 时路由错误无法补救）或专家数相对数据过多（训不透）。

**案例 3：改用 GQA 后训练正常，但某些下游任务掉点明显**

GQA 的质量损失不均匀——需要精细 KV 区分的任务（如长程精确检索）更敏感。若掉点超预期：确认 G 不要太小（G=8 通常安全，G=1 的 MQA 风险高）；或用"从 MHA 权重初始化 GQA"（KV 头做均值/复制）再继续训练（uptraining），比从头训 GQA 质量更好——这是 LLaMA-2 的实际做法。

## 本章小结与第二篇回顾

现代 LLM = miniGPT 骨架 + 一组针对规模化痛点的零件替换：GQA（省 KV cache）、SwiGLU（强 FFN）、MoE（参数算力解耦）、RoPE 插值（长上下文）、稳定性 trick（防大规模发散）。每个改动都是一笔明确的成本收益账，不是玄学。
