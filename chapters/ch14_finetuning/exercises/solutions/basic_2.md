# 基础 2 参考答案：LoRA 参数量账

模型：7B，d=4096，L=32 层。只对**注意力的 q/v 投影**加 LoRA。

**每层 q/v 投影的全参数量**：
- q_proj: d×d = 4096² ≈ 16.8M
- v_proj: d×d = 4096² ≈ 16.8M
- 每层 q+v = 33.6M，32 层 = 1.07B（这是 q/v 投影的全参，不是整个模型的 7B）

**LoRA(r) 对 q/v 的可训练参数**：
每个投影的 LoRA = 2rd（A 是 r×d，B 是 d×r）。q+v 两个投影，L 层：
```
LoRA 参数 = L × 2 投影 × 2rd = 32 × 2 × 2 × r × 4096 = 1,048,576 × r
```

| r | LoRA 可训练参数 | 占全模型 7B | 占 q/v 全参 1.07B |
|---|---|---|---|
| 8 | 8.4M | 0.12% | 0.78% |
| 16 | 16.8M | 0.24% | 1.57% |
| 32 | 33.6M | 0.48% | 3.14% |
| 64 | 67.1M | 0.96% | 6.28% |
| 128 | 134M | 1.92% | 12.6% |

**验证"1% 参数"的说法**：r=16 时可训练参数占整个 7B 模型的 **0.24%**——
远小于 1%。即便 r=64 也只有约 1%。所以"LoRA 用 <1% 参数微调"是准确的，
甚至偏保守（常见配置 r=8~16 只用 0.1~0.3%）。

**关键理解（14.2-②）**：省的不只是这 0.24% vs 100% 的可训练参数比例，
更是**冻结的 99.76% 主干不产生梯度和优化器状态**：
- 全参微调：7B × 16 字节/参数 = 112GB（第 2 章）
- LoRA：冻结主干 7B × 2 字节（只存权重）+ 16.8M × 16 字节（旁路的完整状态）
  ≈ 14GB + 0.27GB ≈ 14.3GB
显存从 112GB 降到 ~14GB——这才是 LoRA 让消费级卡能微调的根本原因。
（QLoRA 再把那 14GB 的主干量化到 4-bit ≈ 3.5GB，见 14.2-③。）

**加分理解**：若对 q/k/v/o + FFN 全部加 LoRA，参数量翻几倍但仍 <5%，
效果更接近全参微调——这是"加多少层 LoRA"的权衡（易错点③）。
