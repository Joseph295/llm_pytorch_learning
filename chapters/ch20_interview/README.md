## 20.3 高频"深度"问题的加分答法

面试区分普通和优秀的，往往是这些"再深一层"的追问。每个都在教程里有伏笔：

- **"为什么是这个数字？"** — 12Ld² 参数量（8）、16 字节/参数（2）、8/3 FFN 宽度（10）、6N FLOPs/token（11）——能现场推导，不是背结论。
- **"大模型和小模型有什么不同？"** — β₂ 配置（9/15 真实教训）、warmup 长度、过拟合 vs 欠拟合（9）、并行策略随规模变（13）——体现你理解"规模"这个变量。
- **"这个技术的代价是什么？"** — 任何优化都有代价：KV cache 换显存（16）、gradient checkpointing 换算力（11）、量化换精度（16）、ZeRO 换通信（13)——展示工程权衡意识。
- **"你怎么排查 X？"** — 系统的排查树而非碰运气：过拟合单 batch 黄金测试（6）、找谁没到齐（12）、显存四类定位（11）、二分定位第一枚 NaN（15）——体现实战方法论。

大多数转大模型的人从算法侧来，缺系统工程能力。你反过来——**分布式系统是你的护城河**。面试时主动关联：

|---|---|---|

| PagedAttention | 操作系统虚拟内存/分页 | 17 |
| continuous batching | 动态任务调度 | 17 |

## 20.5 三个里程碑项目的讲法（作品集）

面试让你"介绍一个项目"时，这三个是弹药。每个都要能讲：做了什么 → 为什么这么做 → 遇到什么问题怎么解决 → 学到什么。

**miniGPT 从零预训练（第 9 章）**：
- 做了什么：自写 BPE tokenizer + 离线数据管线 + 从随机权重训练能续写的 GPT
- 深度点：训练不稳定的完整排查（β₂=0.95 对小模型是毒药、best-checkpoint 作为工程保险）——**这个真实 debug 故事最能体现工程深度**
- 关联：预训练全流程、训练循环、数据工程

**云端 QLoRA 微调 7B（第 15 章）**：
- 做了什么：租云 GPU，QLoRA 把 base 模型微调成 instruct 模型
- 深度点：显存菜单（从量化到 offload）、SFT 掩码、训练疑难杂症手册
- 关联：微调工程、显存管理、云端实战

**mini-vLLM（第 17 章）**：
- 做了什么：手写 continuous batching + PagedAttention 的推理引擎
- 深度点：用操作系统虚拟内存思想解决 KV cache 碎片、token 级动态调度
- 关联：推理优化、系统设计、你的分布式背景
